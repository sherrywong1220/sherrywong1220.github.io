《图解HTTP》学习笔记（六）——HTTP首部

  • HTTP
  • TCP/IP

章节回顾

  • 首部字段给浏览器和服务器提供报文主体大小、所使用的语言、认证信息等内容。
  • 其实在学习笔记（三）中已经简单讲解了什么是HTTP报文，什么是报文主体和报文首部，以及首部的构成。
  • 大家可以先回顾一下《图解HTTP》学习笔记（三）——HTTP报文内的HTTP信息的内容。

首部字段

4种首部字段类型

  • 通用首部字段
    请求报文和响应报文两方都会使用的首部。
  • 请求首部字段
    从客户端向服务器端发送请求报文时使用的首部。补充了请求的附加内容、客户端信息、响应内容相关优先级等信息。
  • 响应首部字段
    从服务器端向客户器端发送请求报文时使用的首部。补充了响应的附加内容也会要求客户端附加额外的内容信息。
  • 实体首部字段
    针对请求报文和响应报文的实体部分使用的首部。补充了资源内容更新时间等与实体有关的信息。

HTTP/1.1首部字段表

  • 大家可以到这个页面浏览：HTTP/1.1首部字段一览 (其中有一个小错误，通用首部字段第一个应该是Cache-Control而不是Cache)

非HTTP/1.1首部字段

  • Cookie、Set-Cookie和Content-Disposition等在其他RFC中定义的首部字段。
  • 这些非正式首部字段统一归纳在RFC4299 HTTP Header Field Registrations中。

End-to-end首部和Hop-by-hop首部

  • 端到端首部(End-to-end Header)
    分在此类别中的首部会转发给请求/响应对应的最终接受目标，且必须保存在由缓存生成的响应中，另外规定它必须被转发。
  • 逐跳首部(Hop-by-hop Header)
    分在此类别的首部只对单次转发有效，会因为通过缓存或代理而不再转发。HTTP/1.1之后的版本，如果要使用逐跳首部，需提供Connection首部字段。
